# Developer Portal

Endpoints principais:

- `GET /api/developer/overview`
- `GET /api/developer/escopos`
- `GET /api/developer/api-keys`
- `POST /api/developer/api-keys`
- `POST /api/developer/api-keys/{id}/revogar`
- `POST /api/developer/api-keys/{id}/rotacionar`
- `GET /api/developer/api-keys/{id}/logs`

O portal deve explicar que API Keys servem para integrações B2B, que a chave só é exibida uma vez e que revogação/rotação são auditadas por tenant.
