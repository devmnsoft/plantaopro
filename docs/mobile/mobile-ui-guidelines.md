# Mobile UI Guidelines (Área Médica e Operacional)

## Princípios
- Mobile-first com componentes touch-friendly.
- Cards com ações principais destacadas.
- Estados de carregamento, vazio e erro em todas as telas.
- Mensagens objetivas com linguagem operacional.

## Componentes obrigatórios
- Header fixo com contexto da tela.
- Cards de resumo (KPI).
- Badges por status (convite, escala, pagamento).
- Modal de confirmação para ações sensíveis.
- Toast para feedback de sucesso/erro.

## Acessibilidade e legibilidade
- Alvos de toque com tamanho mínimo recomendado.
- Contraste de cores para status críticos.
- Ícones sempre acompanhados de rótulo textual.

## Padrões por fluxo
- Convites: aceitar/recusar em destaque e prazo visível.
- Plantões: conflito e disponibilidade visíveis antes da ação.
- Pagamentos: linha do tempo com status financeiro.
- Notificações: contador de não lidas e marcação em lote.
