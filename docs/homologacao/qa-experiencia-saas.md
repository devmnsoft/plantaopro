# QA de experiência SaaS

## Itens revisados

- Layout autenticado com identidade SaaS B2B.
- Menus agrupados por perfil e contexto.
- Topbar com busca, tenant, plano, ambiente, notificações e upgrade.
- Dashboards por perfil ou redirecionamento para home adequada.
- White label com preview visual.
- Planos, módulos e bloqueios com mensagem comercial.
- Matriz de permissões com teste visual de acesso.

## Limitação da rodada

O container não possui `dotnet`; portanto o QA técnico de build/teste deve ser executado em ambiente de desenvolvimento com .NET SDK instalado.
