# Critérios de “Pronto para Venda”

Data de referência: **26/05/2026**

## Produto
- Fluxo comercial funcional (onboarding completo).
- Fluxo operacional funcional (plantão → escala → pagamento).
- Área do médico com experiência aceitável em celular.
- Suporte e ajuda disponíveis no produto.

## Técnica
- Build API e Web sem erros no pipeline.
- Sem artefatos indevidos versionados.
- Auditoria em ações críticas.
- Logs estruturados e monitoráveis.

## Segurança
- Perfis e permissões revisados por módulo.
- JWT validado para Web/API/Mobile.
- Mensagens amigáveis sem stack trace para usuário final.

## Comercial
- Plano, assinatura e faturamento com regras mínimas.
- Demo guiada com roteiro repetível.
- Material de implantação e homologação atualizado.

## Operação
- Roteiro de testes MVP executável.
- Responsáveis por suporte e CS definidos.
- Pendências pós-MVP registradas e priorizadas.
